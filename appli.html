LINEAR SEARCH

Linear search (sequential search) can be applied for searching a particular log in the phonebook (or) word in a dictionary.


TIME COMPLEXITY ANALYSIS:

BEST CASE:

O(1) if the element is found at 1st position i.e. only 1 comparison is required.

AVERAGE CASE:

O(n) irrespective of the size of the array.

WORST CASE:

O(n) if the element is found at last position i.e. n comparisons are required.

BINARY SEARCH

Binary search can be applied for debugging a linear piece of code or figuring out resource requirements for a large system or to ind values in sorted collection or in semiconductor test programs.


TIME COMPLEXITY ANALYSIS:

BEST CASE:

O(1) if the element is found at the mid position i.e. only 1 comparison is required.

AVERAGE CASE:

O(log n) irrespective of the size of the array.

WORST CASE:

O(log n) when n comparisons are required.
